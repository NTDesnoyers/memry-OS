Build a single-user Ninja Selling Operating System with the following exact requirements and constraints.

⸻

PRODUCT GOAL
Create a personal Ninja Selling app that unifies Weekly Meeting Agenda (WMA), FORD & Record relationship tracking, business execution tracking, and client intelligence. The app is the source of truth. Cloze is a downstream mirror for relationship memory, reminders, and follow-ups.

⸻

CORE PRINCIPLES (HARD CONSTRAINTS)
	•	Ninja Selling workflows take priority over CRM conventions
	•	One data entry → many uses
	•	Flat, queryable database schema
	•	Optimized for weekly use over many years
	•	<10 minutes to complete Weekly Meeting Agenda
	•	<60 seconds to log a FORD interaction
	•	No multi-user features
	•	No public access
	•	No gamification
	•	No notification spam
	•	No pipeline UI

⸻

APP MODULES (MUST EXIST)
	1.	PEOPLE (CLIENT INTELLIGENCE CORE)
	2.	FORD & RECORD (RELATIONSHIP LOGGING)
	3.	WEEKLY MEETING AGENDA (WMA)
	4.	BUSINESS TRACKER
	5.	REAL ESTATE REVIEW PLANNER
	6.	DEALS & POST-CLOSING FOLLOW-UP
	7.	CLOZE SYNC (DOWNSTREAM)

⸻

MODULE BEHAVIOR REQUIREMENTS

WEEKLY MEETING AGENDA
	•	One report per calendar week
	•	Allow backdated entries
	•	Pre-fill last week’s values by default
	•	Auto-calculate weekly FORD count from interactions
	•	Display historical trends

FORD & RECORD
	•	One interaction can optionally create a FORD record
	•	Voicemail counts as an interaction
	•	Multiple interactions per person per week allowed
	•	FORD interactions auto-roll up into WMA ford_count

HOT / WARM LIST
	•	Derived from people.status
	•	Display last interaction date
	•	Require a reason for hot/warm status
	•	Require a next action field

REAL ESTATE REVIEWS
	•	Must link to a person
	•	Must support prep + outcome notes
	•	Can auto-generate follow-ups

POST-CLOSING FOLLOW-UP
When a deal status changes to “closed”, auto-create followups at:
	•	7 days
	•	30 days
	•	90 days
	•	6 months
	•	12 months

⸻

CLOZE INTEGRATION (DOWNSTREAM ONLY)
	•	App is source of truth
	•	Cloze receives:
	•	Contact info
	•	Tags
	•	Interaction summaries
	•	Follow-up tasks
	•	No data flows from Cloze back into app
	•	Sync should be idempotent

⸻

ANALYTICS REQUIREMENTS (PHASE 2 READY)

System must support queries such as:
	•	Habit ratings vs closings over time
	•	FORD count vs production
	•	Time/energy rating vs output
	•	People with no interaction in X days
	•	Best performing weeks by activity mix
