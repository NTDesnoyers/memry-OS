3Ô∏è‚É£ EXPERIENCE (MEANING STORAGE)

CREATE TABLE experience (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  person_id UUID REFERENCES person(id) ON DELETE CASCADE,
  type experience_type NOT NULL,
  summary TEXT NOT NULL,
  emotional_valence emotional_valence,
  magnitude_score INT CHECK (magnitude_score BETWEEN 1 AND 5),
  occurred_at TIMESTAMP,
  acknowledged BOOLEAN DEFAULT FALSE,
  logged_at TIMESTAMP DEFAULT NOW()
);

Non-negotiable rule

6Ô∏è‚É£ ACTION INSTANCE (VALUE CREATION)

8Ô∏è‚É£ CRITICAL MVP VIEWS (COPY THESE)

üî• Awe Opportunities Queue

CREATE VIEW awe_opportunities AS
SELECT
  e.id AS experience_id,
  p.id AS person_id,
  p.name,
  e.summary,
  e.magnitude_score,
  e.logged_at
FROM experience e
JOIN person p ON e.person_id = p.id
WHERE e.magnitude_score >= 3
AND e.acknowledged = FALSE
ORDER BY e.logged_at ASC;

This is your highest-leverage UI surface.

üìà What Actually Deepens Relationships

CREATE VIEW action_effectiveness AS
SELECT
  a.type,
  AVG(o.relationship_delta) AS avg_delta,
  COUNT(*) AS sample_size
FROM outcome o
JOIN action_instance ai ON o.action_instance_id = ai.id
JOIN action a ON ai.action_id = a.id
GROUP BY a.type;

This becomes your moat over time.
